Part 1 — Variables & Data Types

~25 min · Q1

1.1 — Variables as Labeled Containers

Start with the analogy: a variable is a sticky note attached to a box. The note has a name (the variable name) and the box holds a value. The note can be moved to a new box (reassignment). Python remembers where the note is pointing.

product_name = "Widget Pro" # str price = 49.99 # float quantity = 12 # int in_stock = True # bool
⭐
Quiz Q1 Verbatimprint(type(3.14)) → <class 'float'>. The answer is c) float. 3.14 has a decimal point so Python infers float. Common wrong answer: int. Demonstrate type() on all four types live — str, int, float, bool.

Part 2 — Output & f-strings

~15 min · Q2
⭐
Quiz Q2 Verbatim — Must Know Coldname="Taylor", price=9.99. print(f"{name} costs ${price}") → "Taylor costs $9.99". Teach students to read the f-string from left to right: f prefix, open quote, literal text, {variable} substitution, more literal text, close quote. The $ before {price} is a literal dollar sign — it's not part of the f-string syntax.
name = "Taylor" price = 9.99 print(f"{name} costs ${price}") # Q2: Taylor costs $9.99
⚠️
Common f-string errorsForgetting the f prefix (just a regular string — no substitution). Forgetting curly braces (prints the variable name literally). Using parentheses instead of curly braces. Show all three errors live.

Part 3 — Operators & Business Formulas

~25 min · Q3, Q4, Q5, Q6
⭐
Quiz Q3, Q4, Q5, Q6 — All Live Code TracesQ3: price=50, qty=3 → revenue=150 (multiplication). Q4: 17//5 = 3 (floor division — drops decimal). Q5: 17%5 = 2 (remainder: 17 = 3×5 + 2). Q6: cost=40, price=60, margin=(60-40)/60 = 20/60 = 0.333. Write these on the board. Have students trace before running.
