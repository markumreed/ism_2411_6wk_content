Rubric Summary (25% Each)

  • Code Clarity: Notebook runs Restart+Run All. Meaningful variable names. Markdown in every section.
  • Visualization Quality: All 3 chart types (bar+line+histogram). Title, x-label, y-label on every chart. Saved to charts/.
  • Business Insight Quality: Findings state specific numbers ("North: $31,200 -- 26% ahead of East"). Written interpretation. At least 1 recommendation.
  • Communication: 5-7 min. Covers dataset, cleaning, 3 findings, recommendation. Code walkthrough. Peer feedback submitted.

Presentation Tips

  • Randomize presentation order. Allocate 8 min per student (7 + 1 transition).
  • If a student says "North had the most revenue" without a number: "Can you give us the specific value?"
  • Model peer feedback for first 2-3 presenters. Be specific: "Your bar chart sorted descending made North's lead immediately obvious."
  • Common issue: slides say "I cleaned the data" without explaining the decisions. Prompt: "What did you find when you ran isnull().sum()?"
  • Common issue: notebook doesn't run cleanly. Confirm at submission: "Did you test Restart + Run All?"

Course Wrap-Up (Last 10 min)

  • Their GitHub repo is now public. Tell them to add the URL to LinkedIn and resume today.
  • ISM3232 (Business Intelligence) builds directly on this foundation. SQL feels like groupby.
  • Optional DataCamp #2 (Intermediate Python) and any courses they haven't done remain available.
  • The skill decays without practice. Encourage one personal project before ISM3232.
