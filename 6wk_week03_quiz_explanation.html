Q01Code Trace — list operations
inventory = ["apples","bananas"]
inventory.append("oranges")
inventory.remove("bananas")
print(inventory)
  • a["apples","bananas","oranges"]
  • b["apples","oranges"]
  • c["bananas","oranges"]
  • dError
Explanation

Start: ["apples","bananas"]. append("oranges") → ["apples","bananas","oranges"]. remove("bananas") → ["apples","oranges"]. Verbatim Q1.

Q02Code Trace — dict access
customer = {"name": "Alice", "age": 30}
print(customer["name"])
  • aAlice
  • bname
  • c{"name":"Alice"}
  • dError
Explanation

customer["name"] accesses the value for key "name" → "Alice". Verbatim Q2 (Week 11).

Q03MC — csv module
Which Python module reads and writes CSV files without installing anything?
  • afile
  • bcsv
  • cpandas
  • dio
Explanation

The csv module is built into Python — no installation needed. pandas is a third-party library. Verbatim Q3 (Week 12).

Q04MC — tuple immutability
What is the key difference between a list and a tuple?
  • aTuples hold more items
  • bLists use {}; tuples use []
  • cTuples are immutable; lists are mutable
  • dLists hold only strings
Explanation

Tuples are immutable — once created, they cannot be modified. Lists are mutable — you can add, remove, or change items. Verbatim Q4.

Q05MC — relative paths
Why use relative file paths (data/sales.csv) instead of absolute paths (/Users/john/Desktop/data/sales.csv)?
  • aShorter to type
  • bWork on any machine regardless of where the project is stored
  • cRun faster
  • dRequired by the csv module
Explanation

Relative paths work on any machine. Absolute paths break when the project moves to a different computer or when another person clones the repo. Verbatim Q5.

Q06Code Trace — dict .get()
customer = {"name": "Alice", "age": 30}
print(customer.get("email", "Not found"))
  • aNone
  • bError
  • c"Not found"
  • d"email"
Explanation

.get(key, default) returns the value if the key exists, or the default if it does not. "email" is not in the dict → returns "Not found". Verbatim Q6.

Q07Code Trace — dict iteration
inventory = {"apples": 10, "bananas": 5}
for item in inventory:
print(item)
  • a10 5
  • bapples bananas
  • c{"apples":10}
  • dError
Explanation

Iterating a dict returns its KEYS, not values. To get values: for v in inventory.values(). To get both: for k,v in inventory.items(). Verbatim Q7.

Q08Code Trace — list len
coords = (40.7128, -74.0060)
print(len(coords))
  • a1
  • b3
  • c2
  • dError
Explanation

The tuple has 2 elements: 40.7128 and -74.0060. len() returns 2. Verbatim Q8.

Q09MC — dict method
Which dict method returns all key-value pairs as tuples?
  • a.keys()
  • b.values()
  • c.items()
  • d.pairs()
Explanation

.items() returns a view of (key, value) tuples — used in for k,v in d.items() loops. .pairs() does not exist. Verbatim Q9.

Q10MC — csv DictReader
When using csv.DictReader to read a CSV, what type is each row?
  • aA list
  • bA tuple
  • cA dict
  • dA string
Explanation

DictReader reads each row as a dict where keys are the header column names. This is why you write row["price"] instead of row[2].
