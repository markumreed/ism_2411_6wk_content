Part 1 · Variables

Variables — Labeled Containers

A variable is a name that points to a value stored in memory

# Assignment: name = value product_name = "Widget Pro" price = 49.99 quantity = 12 in_stock = True # Variable names: lowercase_with_underscores # NOT: ProductName, not: product-name # Reassignment: the label moves to a new value price = 44.99 # price is now 44.99 price = price + 5 # price is now 49.99
🔑
= is assignment, not equalityx = 5 means "point x at the value 5." It does NOT mean "x equals 5 forever." The variable can be reassigned at any time. For equality checking, use == (two equals signs).
# x = 5 means: put 5 in a box named x x = 5 x = x + 3 print(x) # → 8 # == checks equality (returns True/False) print(x == 8) # → True

Part 1 · Data Types

The Four Core Data Types

Python automatically infers the type — use type() to check

# str — text, always in quotes name = "Alice" city = 'Tampa' # single or double quotes # int — whole numbers, no decimal quantity = 12 year = 2025 # float — numbers with decimal price = 49.99 margin = 0.35 # bool — True or False (capital T/F) in_stock = True is_sale = False
# Quiz Q1 verbatim: print(type(3.14)) <class 'float'> # type() tells you what kind of data print(type("hello")) # str print(type(42)) # int print(type(True)) # bool # Common trap: "True" is a str, True is a bool a = True # ← bool b = "True" # ← str (not the same!)

Part 2 · Comparison & Logic

Comparison & Logical Operators

Produce True/False — the foundation of all decision-making code

# Comparison operators → bool x = 10 y = 3 print(x > y) # True print(x == y) # False (== not =) print(x != y) # True print(x >= 10) # True # Logical operators: and, or, not score = 85 print(score > 80 and score < 90) # True print(score > 90 or score > 80) # True
# Business context: is order big enough? order_value = 250 is_premium = order_value >= 200 print(is_premium) # True # Qualifies for free shipping AND discount? ships_free = order_value > 100 gets_disc = order_value > 200 print(ships_free and gets_disc) # True
⚠️
= vs === is assignment (set a value). == is comparison (check equality). This is the #1 beginner error. x = 5 puts 5 in x. x == 5 returns True or False.

Part 3 · Type Conversion

Converting Between Types

Python won't automatically convert — you must be explicit

# Explicit conversion functions int("42") # "42" → 42 (string to int) float("9.99") # "9.99" → 9.99 (string to float) str(100) # 100 → "100" (int to string) # input() ALWAYS returns a string qty_str = input("Enter quantity: ") qty = int(qty_str) # convert before math! # Or in one line: qty = int(input("Qty: "))
# Type mismatch crash — common bug price_str = "49.99" # revenue = price_str * 3 ← TypeError! revenue = float(price_str) * 3 # ✓ print(revenue) # 149.97 # Check with type() print(type(price_str)) # str print(type(float(price_str))) # float
💡
This matters for pandasWhen you read CSV files (Week 4), every column starts as a string. You'll convert them explicitly — int(row["qty"]) and float(row["price"]). Understanding type conversion now makes Week 4 make sense.

Business Context

Business Formulas in Python

Every business metric is just arithmetic with variables — now you can automate it

# Retail pricing formulas cost = 40.00 price = 65.00 quantity_sold = 150 revenue = price * quantity_sold profit = (price - cost) * quantity_sold margin = (price - cost) / price print(f"Revenue: ${revenue:,.2f}") print(f"Profit: ${profit:,.2f}") print(f"Margin: {margin:.1%}")
Revenue: $9,750.00 Profit: $3,750.00 Margin: 38.5% # These 6 lines replace an Excel worksheet # Change cost to 45: re-run → instant update # Scale to 10,000 products: same code # Add to GitHub: team uses same formula
🎯
Why This MattersEvery data analysis job requires computing metrics: revenue, margin, growth rate, conversion rate. These are arithmetic. Python makes them reproducible and scalable. Week 1 formulas become Week 4 pandas columns.

Week 1 Traps — Know These Before the Quiz

Using = instead of == in comparisons

Forgetting quotes around strings
name = Alice # NameError name = "Alice" # ✓
Math on strings instead of numbers
price = "49.99" total = price * 3 # TypeError! total = float(price) * 3 # ✓
// vs / confusion
