Q01MC — type() check
What is the output of: print(type(3.14))?
  • aint
  • bstr
  • cfloat
  • dbool
Explanation

float — 3.14 has a decimal point so Python infers float type. Verbatim Q1.

Q02Code Trace — f-string output
What is the output?
name = "Taylor" price = 9.99 print(f"{name} costs ${price}")
  • aTaylor costs $9.99
  • bname costs $price
  • cTaylor costs 9.99
  • dError
Explanation

f-strings substitute variable values. {name} → Taylor, {price} → 9.99. The $ is a literal character. Output: Taylor costs $9.99. Verbatim Q2.

Q03Code Trace — multiplication
price = 50 quantity = 3 revenue = price * quantity print(revenue)
  • a500
  • b53
  • c150
  • dError
Explanation

50 * 3 = 150. Verbatim Q3.

Q04MC — floor division
What is the result of 17 // 5?
  • a3.4
  • b2
  • c3
  • d4
Explanation

// is integer (floor) division — drops the decimal. 17 ÷ 5 = 3 remainder 2. Result: 3. Verbatim Q4.

Q05MC — modulo
What is the result of 17 % 5?
  • a3.4
  • b2
  • c3
  • d4
Explanation

% returns the remainder. 17 = 3×5 + 2. Remainder = 2. Verbatim Q5.

Q06Code Trace — margin formula
cost = 40 price = 60 margin = (price - cost) / price print(margin)
  • a0.25
  • b0.5
  • c0.333...
  • d20
Explanation

(60-40)/60 = 20/60 = 0.3333333... Python true division preserves the decimal. Verbatim Q6.

Q07MC — bool type
Which of the following is a boolean value in Python?
  • a'True'
  • bTrue
  • c1.0
  • d"false"
Explanation

True (capital T, no quotes) is a bool. 'True' with quotes is a str. 1.0 is a float.

Q08Code Trace — reassignment
x = 5 x = x + 3 print(x)
  • a5
  • b3
  • c8
  • dError
Explanation

x starts at 5. x = x + 3 evaluates the right side first: 5 + 3 = 8. Then assigns 8 back to x. Output: 8.

Q09MC — int vs float
What type does Python assign to the value 42?
  • astr
  • bfloat
  • cint
  • dbool
Explanation

42 has no decimal point so Python infers int. 42.0 would be float.

Q10Code Trace — type conversion
price_str = "49.99" revenue = float(price_str) * 3 print(revenue)
  • aTypeError
  • b149.97
  • c"49.99" * 3
  • dError
Explanation

float("49.99") converts the string to 49.99. 49.99 * 3 = 149.97. Without float(), multiplying a string by an int repeats it — which would raise a TypeError for non-integer multiplication.
